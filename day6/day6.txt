보안 엔지니어 기본 단계
1) 감지 (Detection)
- IDS/IPS 알림
- 서버 CPU 급증
- 관리자 계정 로그인 시도 증가
- 웹 서버 로그 대량 증가

2) 초기 분석 (Triage)
- 공격 흔적이 있는지 가장 빠르게 확인
- 프로세스/로그/네트워크 연결 상태 확인

3) 격리 (Containment)
- 침해된 서버를 내부망에서 분리
- 서비스 중단 (필요 시)

4) 정밀 조사 (Analysis)
-조사 대상:	/var/log/auth.log
		/var/log/syslog
- 악성 파일 해시 확인
- 공격자가 심어놓은 백도어/스케줄러

5) 복구 (Recovery)
- 설정 복구
- 패치 적용
- 계정 초기화

6) 재발 방지 (Lessons Learned)
- 방화벽 규칙 강화
- root 로그인 막기
- 로그 보관/전송 설정
